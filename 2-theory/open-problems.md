# Open Problems

Open problems are tracked as **`OPN-*` traceability claims** — the canonical, versioned registry — and surfaced three ways:

- **Registry view (generated):** [`../traceability/generated/open-problems.md`](../traceability/generated/open-problems.md) — the full table with confidence, dependencies, statements, and failure conditions, built from the claims by `build.py` (do not hand-edit).
- **Active subset (issues):** the problems currently being worked carry a GitHub issue labelled [`open-problem`](https://github.com/jdlongmire/triadic-reality-theory/issues?q=is%3Aissue+is%3Aopen+label%3Aopen-problem), each with `Tracks: OPN-id` in its body and `tier:` / `conf:` labels.
- **Source of truth:** [`../traceability/claims/`](../traceability/claims/) (`OPN-*.yaml`).

The tracking method is described in [`../0-program-methods/github-workflow.md`](../0-program-methods/github-workflow.md#open-problem-tracking).

**Keystone:** [`00-foundational/actualization-resolution.md`](00-foundational/actualization-resolution.md) (claim `OPN-002`) is the shared root of the Born-measure (`OPN-004`) and gravity-fork problems — whether *A* registers obstruction (linear → GR) or best-fit projects (quadratic → modified gravity).
