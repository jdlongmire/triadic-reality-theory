# Session journal

One dated entry per working session — the programme's development history and the
Lakatosian problemshift record. Entries travel with the repo (durable, versioned).

**Bracket each session** with the lifecycle scripts ([github-workflow.md](../github-workflow.md#session-lifecycle)):

- start: `python3 0-program-methods/ops/research-start.py` (orientation briefing)
- wrap: `python3 0-program-methods/ops/research-wrap.py` (hygiene gate) — then add the entry below

**Convention:** `YYYY-MM-DD[-tag].md`. Capture: objectives/tasks touched, what confirmed,
what was *recorded as a refutation*, claims/issues changed, and a **Carry to next**
section (which `research-start` surfaces on the next session).
